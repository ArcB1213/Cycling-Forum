# 自行车数据论坛系统项目总结

## 一、项目概述

本项目是一个面向自行车运动爱好者的综合性数据平台，集成了自行车赛事数据展示、车手评价系统和论坛讨论功能。系统采用前后端分离架构，使用现代Web技术栈构建，具有高性能、高并发、易扩展的特点。

### 1.1 项目核心价值

- **数据可视化**：提供完整的自行车赛事数据展示，包括赛事、赛段、车手、车队信息
- **互动评价系统**：用户可以对车手进行评分和评价，形成车手口碑
- **实时论坛功能**：支持多级嵌套评论，实时更新，让车迷可以自由讨论
- **用户认证体系**：完整的注册、登录、邮箱验证、密码找回功能

### 1.2 技术架构亮点

- **全异步架构**：FastAPI + SQLAlchemy 2.0 + aiomysql
- **缓存优化**：Redis写回策略，支持高频访问
- **实时通信**：WebSocket实现评论实时更新
- **性能优化**：批量查询优化，数据库索引优化
- **类型安全**：TypeScript + Pydantic严格类型定义

**内存构建**：
```python
# 递归构建评论树
def build_comment_tree(comments: List[ForumComment]) -> Dict[int, List[ForumComment]]:
    tree = defaultdict(list)
    for comment in comments:
        if comment.parent_id:
            tree[comment.parent_id].append(comment)
        else:
            tree[comment.root_id].append(comment)
    return tree
```

#### 4.1.2 写回缓存策略

**Redis计数器**：

```python
# 5分钟批量写入MySQL
def write_back_views():

    for post_id, count in views.items():
        db.execute(
```

```python
class CacheManager:
    def __init__(self):
        self.redis = aioredis.Redis.from_url(REDIS_URL)
        self.memory_cache = {}  # LRU缓存

    async def get(self, key: str):
        # 先查内存缓存
        if key in self.memory_cache:
            return self.memory_cache[key]

        # 再查Redis
        value = await self.redis.get(key)
        if value:
            self.memory_cache[key] = value
            return value

        return None
```
