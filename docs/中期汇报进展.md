## 项目功能总结

### 三、数据处理与后端 API

#### **已实现的 API 端点：**

| 端点                           | 方法 | 功能                 | 返回数据                           |
| ------------------------------ | ---- | -------------------- | ---------------------------------- |
| `/api/races`                   | GET  | 获取所有赛事         | 赛事列表                           |
| `/api/editions/<race_id>`      | GET  | 获取某赛事的所有届数 | 届数列表（按年份降序）             |
| `/api/stages/<edition_id>`     | GET  | 获取某届的所有赛段   | 赛段列表                           |
| `/api/results/<stage_id>`      | GET  | 获取某赛段的成绩     | 成绩列表（按排名升序）             |
| `/api/riders`                  | GET  | 获取所有车手         | 车手列表                           |
| `/api/riders/<rider_id>`       | GET  | 获取车手详情+统计    | 车手信息、参赛数、冠军数、车队列表 |
| `/api/riders/<rider_id>/races` | GET  | 获取车手所有参赛记录 | 完整参赛记录表                     |
| `/api/riders/<rider_id>/wins`  | GET  | 获取车手冠军记录     | 冠军赛段详情列表                   |

#### **数据库查询优化：**

- 使用 SQLAlchemy ORM 进行联表查询
- `JOIN` 关联 Race、Edition、Stage、StageResult、Rider、Team 表
- 按年份、赛段号、排名排序
- 使用 `DISTINCT` 去重统计车队数量
