### 4. 性能优化技术

- ✅ 使用 `selectinload()` 预加载关联数据（避免 N+1 查询）
- ✅ 使用 `run_in_executor()` 处理 CPU 密集型操作（密码哈希）
- ✅ 批量查询优化（如获取多个车队信息）
- ✅ 智能缓存策略（热数据短过期，冷数据长过期）
- ✅ **异步缓存失效**: 实现 `invalidate_cache_async`，避免在异步请求中调用同步 Redis 操作导致事件循环阻塞
- ✅ **合并数据库查询**: 在评分提交等复杂逻辑中合并查询，减少数据库往返次数
- ✅ **连接池优化**: 增加 `DB_POOL_SIZE` (20) 和 `DB_MAX_OVERFLOW` (40)，并设置 `pool_timeout` (30s)

## 🚀 部署建议

### 开发环境

```bash
# 启动命令
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

### 生产环境

```bash
# 多 worker 进程（推荐）
uvicorn app:app --workers 4 --host 0.0.0.0 --port 8000

# 或使用 gunicorn + uvicorn worker
gunicorn app:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

### 环境变量配置

```bash
# .env
DB_POOL_SIZE=20
DB_MAX_OVERFLOW=40
DB_POOL_RECYCLE=3600
REDIS_HOST=localhost
REDIS_PORT=6379
SECRET_KEY=your-secret-key-here
```

## 💡 经验总结

### 关键成功因素

1. ✅ **预加载关联数据** - 使用 `selectinload()` 避免 N+1 查询问题
2. ✅ **智能缓存策略** - Redis 缓存大幅减少数据库压力
3. ✅ **线程池优化** - CPU 密集型操作不阻塞事件循环
4. ✅ **连接池配置** - 合理的数据库连接池大小
5. ✅ **向后兼容** - 同步和异步端点共存，平滑迁移

### 踩过的坑

1. ❌ 缓存存储协程对象 → ✅ 清除 Redis 缓存解决
2. ❌ 懒加载导致异步错误 → ✅ 使用 `selectinload()` 预加载
3. ❌ 类型注解导致装饰器问题 → ✅ 简化类型注解
4. ❌ N+1 查询性能差 → ✅ 批量查询优化
